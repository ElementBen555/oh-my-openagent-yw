# Oh My OpenAgent

你同时折腾着 Claude Code、Codex、各种奇奇怪怪的开源模型。配工作流。给 Agent 调 Bug。

这些事我们替你做完了。全部测试过。只留下真正跑得起来的代码。

装上 oh-my-openagent。敲 `ultrawork`。就完事了。

## 安装

oh-my-openagent 以同一产品的两个版本发布：

- **Ultimate 版本**（omo for OpenCode）— 完整功能。11 个 Agent、53–60 个生命周期钩子、**7 个内置 MCP**、所有斜杠命令、Team Mode、ulw-loop、ultrawork、hashline 编辑 — 应有尽有。
- **Light 版本**（omo for Codex CLI）— 可移植组件，适配 Codex 插件系统：`rules`、`comment-checker`、`git-bash`、`lsp`、`ultrawork`、`ulw-loop`、`start-work-continuation`、`telemetry`，外加 plugin-scoped MCP（`grep_app`、`context7`、`codegraph`、`git_bash`、`lsp`）和共享 `ast-grep` skill。无 Agent 编排，无 `team_*` 工具。

挑一个（或两个）。

### 快速开始

| 你要的 | 运行 | 写入磁盘的内容 |
| :--- | :--- | :--- |

### 自律军团 (Discipline Agents)

**Sisyphus** (`claude-opus-4-7` / `kimi-k2.6` / `glm-5.1`) 是你的主指挥官。负责制定计划、分配任务并以激进并行策略推动任务完成。他从不半途而废。

**Hephaestus** (`gpt-5.5`) 是你的自主深度工作者。只需要给他目标，不要给他具体做法。他会自动探索代码库，从头到尾独立执行，绝不会中途要你当保姆。

**Prometheus** (`claude-opus-4-7` / `kimi-k2.6` / `glm-5.1`) 是你的战略规划师。通过访谈模式，在动一行代码之前先通过提问确定范围并构建详尽计划。

每个 Agent 都针对底层模型的特点进行了专门调优。[阅读背景设定 →](docs/guide/overview.md)

### Team Mode (v4.0)

一个 Agent 已经够快。一支协调的团队是 *毁灭性* 的。

Team Mode 把 oh-my-openagent 从「带子 Agent 的单个 Agent」升级为真正的多 Agent 系统。领导 Agent 协调一队按类别专业化的成员，全部 **并行** 运行，通过专用工具通信。在 tmux 布局中同时观察每个成员的工作。

```jsonc
// .opencode/oh-my-openagent.jsonc
{
  "team_mode": {
    "enabled": true,
    "max_parallel_members": 4,
    "tmux_visualization": true
  }
}
```

重启 OpenCode，`team_*` 工具家族就会解锁。已有两个技能站在它之上:

- **`hyperplan`** — 5 个敌对 Agent 在写下第一行代码之前从正交角度撕碎你的计划。
- **`security-research`** — 3 个漏洞猎手 + 2 个 PoC 工程师并行审计你的代码库。

> **默认关闭。需要时再开。** [Team Mode 完整指南 →](docs/guide/team-mode.md)

### 智能体调度机制

当 Sisyphus 把任务分配给子智能体时，他选择的不是具体的模型，而是 **类别 (Category)**。系统会自动将类别映射到最合适的模型：

| 类别                 | 作用领域               |
| :------------------- | :--------------------- |
| `visual-engineering` | 前端、UI/UX、设计      |
| `deep`               | 深度自主调研与执行     |
| `quick`              | 单文件修改、修错字     |
| `ultrabrain`         | 复杂硬核逻辑、架构决策 |

### 完全兼容 Claude Code

你已经花了大力气调教好了 Claude Code 的配置？太好了。

这里完美兼容所有的 Hook、命令、技能、MCP 以及插件。

### 赋予 Agent 世界级的开发工具

- **LSP**: 支持重命名、跳转定义、引用查找、诊断。
- **AST-Grep**: 支持 25 种编程语言，理解语法树的模式匹配和代码重写。
- **Tmux**: 真实交互式终端，支持 REPL、调试器、TUI 工具。
- **MCP**: 内置 Web 搜索、官方文档、GitHub 代码搜索、动态反思式推理、GitHub 仓库管理。

### MCP 离线韧性

网络断了？我们不会崩溃。

远程 MCP（`websearch`、`context7`、`grep_app`）在插件初始化时进行 **2 秒 HEAD 探测**。不可达的服务会被自动禁用（`enabled: false`），保证插件在无网环境下继续运行。

### 拒绝瞎改：基于内容哈希的编辑工具

绝大多数所谓的 Agent 故障，其实并不是大模型变笨了，而是他们用的文件编辑工具太烂了。

> "目前所有工具都无法为模型提供一种稳定、可验证的行定位标识……" <br/>- [Can Bölük, The Harness Problem](https://blog.can.ac/2026/02/12/the-harness-problem/)

受 [oh-my-pi](https://github.com/can1357/oh-my-pi) 启发，我们做出了 **Hashline**。Agent 读到的每一行代码，末尾都会打上一个强绑定的内容哈希值：

```
11#VK| function hello() {
22#XJ|   return "world";
33#MB| }
```

Agent 发起修改时必须通过这些标签引用目标行。如果文件在此期间发生变化，哈希验证失败，编辑会被直接驳回。在 Grok Code Fast 1 上，修改成功率从 **6.7% 飙升至 68.3%**。

### 深度上下文初始化：`/init-deep`

执行一次 `/init-deep`。它会生成一套树状的 `AGENTS.md`：

```
project/
├── AGENTS.md              ← 全局级架构与约定
├── src/
│   ├── AGENTS.md          ← src 级规范
│   └── components/
│       └── AGENTS.md      ← 组件级详细说明
```

Agent 自动加载对应 Context。

### 让 Agent 动手前先过脑子：Prometheus

碰到了硬骨头？千万不要扔个 Prompt 就双手合十祈祷。

输入 `/start-work`，召唤 Prometheus 出场。**他会像一个真实的主管那样去采访你**，主动深挖需求、指出模糊地带，并在改动哪怕一行代码之前产出经过严密论证的计划。

### 技能系统 (Skills)

这里的 Skills 绝不只是一段无脑的 Prompt 模板。它们包含了：

- 面向特定领域的极度调优系统指令。
- 按需加载的独立 MCP 服务器。
- 对 Agent 能力边界的强制约束。

默认内置：`playwright`（浏览器自动化）、`git-master`（原子级提交与 rebase 手术）、`frontend`（设计感拉满的 UI 实现）。

想加你自己的？放进 `.opencode/skills/*/SKILL.md` 或者 `~/.config/opencode/skills/*/SKILL.md` 就行。

## 如何卸载

1. **从你的 OpenCode 配置文件中去掉插件**

   编辑 `~/.config/opencode/opencode.json`（或 `opencode.jsonc`），从 `plugin` 数组中删掉 `"oh-my-openagent"` 或旧的 `"oh-my-opencode"` 条目。

2. **删除配置文件（可选）**

   ```bash
   rm -f ~/.config/opencode/oh-my-openagent.jsonc ~/.config/opencode/oh-my-opencode.jsonc
   rm -f .opencode/oh-my-openagent.jsonc .opencode/oh-my-opencode.jsonc
   ```

3. **移除 Codex Light 版本**

   ```bash
   npx lazycodex-ai uninstall
   ```

- **LSP & AST 工具**: 重构、重命名、诊断、语法树搜索
- **基于哈希的编辑**: `LINE#ID` 验证，零错位错误
- **上下文注入**: 自动加载 AGENTS.md、README.md、规则文件
- **Claude Code 兼容**: 完整 Hook 系统、命令、技能、Agent、MCP
- **7 个内置 MCP**: websearch、context7、grep_app、sequential_thinking、github、lsp、codegraph，支持离线健康检查与优雅降级
- **Session 工具**: 会话历史列表、读取、搜索、分析
- **Doctor 命令**: 内置诊断（`bunx oh-my-openagent doctor`）
- **模型回退**: 每个 Agent 独立的后备模型链
- **会话恢复**: 自动从错误、上下文超限、API 失败中恢复

我在个人项目上烧掉了 $24K 的 LLM token。试遍了所有工具。把配置调到死。OpenCode 赢了。

我遇到的每一个问题，修复方案都嵌在这个插件里。装上，跑起来。

这个项目深受 [AmpCode](https://ampcode.com) 和 [Claude Code](https://code.claude.com/docs/overview) 影响。功能被移植，经常被改进。还在持续构建中。它叫 **Open**Code。

我是本项目最狂热的用户：
- 谁的逻辑最锐利？
- 谁是调试之神？
- 谁的文笔最好？
- 谁统治前端？
- 谁霸占后端？
- 日常驾驶谁最快？
- 对手在发什么货？

这个插件是精华的萃取。取最好的。有改进？欢迎 PR。

**别再纠结选哪个 harness 了。我会研究、偷师最好的，然后在这里发出来。**

听起来很狂？有更好的办法？来贡献。欢迎。

与任何提到的项目或模型都没有商业关联。纯粹是个人实验。

致谢：LazyCodex 这个名字的灵感来自 [LazyVim](https://github.com/LazyVim/LazyVim)。Ultragoal、UltraQA 和 Sparkshell 的想法来自 [oh-my-codex](https://github.com/Yeachan-Heo/oh-my-codex)，为 OmO 从概念重新实现。
